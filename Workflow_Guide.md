# 七律空间 · 开发工作流指南

## 一、 核心概念：Git 和 GitHub 到底啥关系？

你可以把它们想象成 **“手机相册”** 和 **“Instagram/朋友圈”** 的关系。

### 1. 本地文件夹 (Workspace)
*   **位置**：`C:\Users\Administrator\Gemini_Playground\PoetrySite`
*   **角色**：这是你的**工作室**。
*   **状态**：这里的文件是“活”的，你随时在修改（装修）。如果不保存，断电就没了。

### 2. 本地 Git (.git 文件夹)
*   **角色**：这是你工作室里的**“时光保险箱”**。
*   **功能**：
    *   它不联网，完全在你的电脑硬盘里。
    *   当你执行 `git commit`（存档）时，它会给你的工作室拍一张“快照”，永久锁进保险箱。
    *   **后悔药**：不管你把工作室砸得多烂，只要保险箱里有存档，随时能一键复原。

### 3. 云端 GitHub (Remote)
*   **角色**：这是**“对外展览馆”**。
*   **功能**：
    *   它用来存放在互联网上，让别人访问（GitHub Pages 就是基于此）。
    *   它还是你的**“异地灾备中心”**。万一你电脑炸了，代码在云端还在。
    *   **同步**：你必须主动把保险箱里的东西“推”（Push）上去，它才会更新。

## 二、 我们刚才经历了什么？（全流程复盘）

我们刚才的操作其实分成了三个阶段：

### 第一阶段：本地装修（写代码）
1.  我们修改了 `index.html` 和 `style.css`。
2.  实现了“网格背景”、“模块化布局”、“页脚致谢”。
3.  **此时状态**：只有你的桌面上是新的，Git 保险箱和 GitHub 云端都还不知道。

### 第二阶段：本地存档（Git Commit）
1.  **`git add .`**
    *   **人话**：把桌上所有改动的文件打包，放在保险箱门口。
2.  **`git commit -m "..."`**
    *   **人话**：把包扔进保险箱，贴上封条（写上备注，比如“巅峰版”），并生成一个唯一的编号（如 `f612b85`）。
3.  **此时状态**：你的电脑里安全了，可以随时回滚，但云端还是旧的。

### 第三阶段：云端发布（Git Push）
1.  **`git remote add origin ...`**
    *   **人话**：在保险箱上装个传送门，目标地址指向你的 GitHub 仓库。
2.  **`git push origin master`**
    *   **人话**：把保险箱里最新的存档，通过传送门“推”到 GitHub 上。
3.  **`git push origin master:main`** (我们最后做的那步)
    *   **人话**：因为 GitHub 默认看 `main` 频道，而我们一直在 `master` 频道工作。这行命令是强行把我们的成果同步到 `main` 频道，确保观众能看到。

以后你只要改了字，或者换了图，只需要对我下这三个指令：

1.  **“兄弟，把改动存一下”**
    *   我会在后台运行：`git add .` 和 `git commit`
2.  **“兄弟，推送到线上”**
    *   我会在后台运行：`git push`
3.  **喝茶验收**
    *   打开 `https://qilv-lattice.github.io/` 查看效果。

## 四、 常见误区
*   **误区**：我改了文件，GitHub 就自动变了。
    *   **真相**：不会。你不 `Push`，GitHub 永远不知道你改了啥。
*   **误区**：Everything 搜不到备份。
    *   **真相**：Git 把文件压缩加密藏在隐藏文件夹 `.git` 里了，只有用 Git 命令才能调取。
